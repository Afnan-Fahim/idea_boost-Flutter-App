1.1 REGION TIER RESOLUTION (ANTI-VPN)
Region tier is determined by the lowest value of:
    • App Store / Play Store country (primary)
    • Device locale (secondary)
    • IP-based region (fallback)
If mismatch is detected → user is downgraded to the lowest tier.
Tier assignment is cached per install and re-evaluated only on app update.

2. GLOBAL RULES (APPLY TO ALL TIERS)
    • AI models:
        ◦ Default → gpt-4.1-nano
        ◦ Premium → gpt-4.1-mini (Tier-1 PRO only)
    • All limits must be configurable via Remote Config
    • Daily counters reset every 24h
    • Hard caps must not be bypassable
    • No unlimited AI access for free users
    • No PII storage or account-based tracking

2.1 OUTPUT LENGTH LIMITS (MANDATORY, SERVER-SIDE)
    • All AI responses must be capped using max_output_tokens (server-side enforced; client values ignored).
    • Limits must be configurable via Remote Config per tier and plan.
Recommended defaults:
    • Tier-1 (non-PRO / rewarded): max_output_tokens = 800
    • Tier-2 (rewarded): max_output_tokens = 600
    • Tier-3 (rewarded): max_output_tokens = 400
    • Tier-1 PRO: max_output_tokens = 1600
For non-PRO users (rewarded/free), responses must be concise and should not exceed the configured cap.
Hard rule: AI generation is executed only after the server applies the token cap.
Context control (cost safety)
    • Limit conversation history sent to the model (e.g., last 6–10 messages)
    • Enforce max user input length (server-side) to prevent excessive input token cost
Remote Config keys (examples):
    • maxOutputTokens_tier1_free
    • maxOutputTokens_tier1_pro
    • maxOutputTokens_tier2
    • maxOutputTokens_tier3
    • maxContextMessages
    • maxUserInputChars

3. TIER-1 MONETIZATION LOGIC
(Business Core)
3.1 FIRST-TIME ONBOARDING (ONE-TIME ONLY)
    • 2 free AI generations
    • No ads
    • No paywall before usage
    • Applies once per device/install
UX: “Try 2 AI scripts for free”
Trial protection
Trial access is protected by device-level fingerprint:
trialKey = hash(device_install_id + app_install_id)
If trialKey has been used before:
    • Trial is blocked
    • User proceeds directly to standard monetization rules
(No PII, store-compliant)

3.2 DAILY USAGE AFTER TRIAL
    • 0 daily free generations
    • AI access via:
        ◦ PRO subscription
        ◦ Rewarded ads

3.3 REWARDED ADS (TIER-1)
    • 1 rewarded ad → unlocks 2 AI generations
    • Max rewarded ads/day: 2
    • Max AI/day (non-PRO): 4
Limit message:
“You’ve reached today’s AI limit. Come back tomorrow or upgrade to PRO.”

3.4 PRO SUBSCRIPTION (TIER-1 ONLY)
Benefits:
    • No ads
    • Unlimited access within caps
    • Access to premium AI model
Soft caps (UI-visible):
    • Nano: 20/day
    • Mini: 20/day
Hard caps (server-side, hidden):
    • Nano: 100/day
    • Mini: 60/day
If hard cap exceeded:
    • AI access temporarily paused
    • Automatic reset after 24h
    • No user-visible penalty messaging

3.5 AI MODEL ROUTING (TIER-1)
IF user.plan == PRO AND quality == premium
→ gpt-4.1-mini
ELSE
→ gpt-4.1-nano

6.1 ABUSE DETECTION (LIGHTWEIGHT)
Signals:
    • Extremely high request frequency
    • High prompt similarity ratio
    • Continuous usage beyond normal session duration
Mitigation:
    • Temporary throttling
    • Forced downgrade to nano model
    • Silent cooldown (no UI penalties)

6.2 ADMOB FAILSAFE MODE
If AdMob fill rate drops below threshold or an account warning is detected:
    • Rewarded ads disabled in Tier-2 and Tier-3
    • AI access temporarily locked for non-PRO users
    • Tier-1 PRO users remain unaffected
